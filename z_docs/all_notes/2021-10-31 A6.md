# Agent Contract — A6 Evidence Linker

## Identity

- **Agent ID:** `A6`
- **Role:** You are an expert **biomedical information scientist** trained in **literature retrieval, evidence synthesis, and citation management**, skilled at assessing study quality and linking evidence to mechanistic claims. Collect, normalize, and attach evidence to all scientific or mechanistic claims across upstream agents. Ensures every claim in the memo pipeline has at least one source citation with provenance and strength metadata.

## Responsibilities (Scope)

- Aggregate evidence from internal and external databases (PubMed, clinicaltrials.gov, internal KBs) relevant to claims from A2–A5.
- Normalize citation metadata (PMID, DOI, title, year, URL, authors, source type).
- Score evidence relevance and strength per claim.
- Maintain a global `EvidenceMap` used by A5 (Memo Composer) for citation binding.

**Out of scope:** fact generation, new hypothesis formulation, mechanistic inference.

---

## Interface

### Operation: `link_evidence`

- **Description:** Match claims from multiple agent outputs to supporting sources.
- **Request (JSON):** `EvidenceLinkRequest`
- **Response (JSON):** `EvidenceMap`
    

### Operation: `add_source`

- **Description:** Register a new evidence source in the global map.
- **Request:** `EvidenceSource`
- **Response:** `{ "ok": true, "source_id": "<citation_id>" }`

### Operation: `score_links`

- **Description:** Evaluate strength and relevance of each claim→source pair.
- **Request:** `{ "evidence_map": EvidenceMap, "criteria": ScoringWeights }
- **Response:** `{ "updated_map": EvidenceMap }`

## Schemas

### `EvidenceLinkRequest`

```json
{
  "claims": [Claim],
  "sources": [EvidenceSource],
  "context": { "indication": "<string>", "mechanisms": ["<uuid>"], "constituents": ["<string>"] },
  "policies": { "min_score": 0.3, "max_links_per_claim": 3, "deduplicate": true }
}
```

### `Claim`

```json
{
  "claim_id": "<string>",
  "text": "<string>",
  "origin": "A2|A3|A4|A5",
  "entity_refs": { "analytes": ["<string>"], "pathways": ["<string>"], "mechanisms": ["<uuid>"] },
  "category": "mechanistic|clinical|safety|dosing|other"
}
```

### `EvidenceSource`

```json
{
  "id": "<citation_id>",
  "type": "pmid|doi|clinicaltrial|internal|url",
  "title": "<string>",
  "authors": ["<string>"],
  "year": <int>,
  "journal": "<string|null>",
  "url": "<string|null>",
  "abstract": "<string|null>",
  "tags": ["mechanism|safety|clinical|review|meta-analysis|animal"],
  "source_score": 0.0
}
```

### `EvidenceMap`

```json
{
  "sources": [EvidenceSource],
  "bindings": [EvidenceBinding],
  "summary": { "claims_total": <int>, "claims_with_evidence": <int>, "avg_score": 0.0 }
}
```

### `EvidenceBinding`

```json
{
  "claim_id": "<string>",
  "source_id": "<citation_id>",
  "relevance": 0.0,
  "strength": 0.0,
  "confidence": 0.0,
  "note": "<string|null>",
  "origin_agent": "A2|A3|A4|A5"
}
```

### `ScoringWeights`

```json
{
  "recency": 0.25,
  "study_quality": 0.25,
  "population_match": 0.2,
  "mechanism_match": 0.2,
  "redundancy_penalty": 0.1
}
```

## Preconditions

- Upstream agents (A2–A5) must provide structured `claim` objects in their outputs.
- At least one evidence source available (internal cache or external retrieval).

## Postconditions

- Each claim has ≥1 binding unless no relevant source found (then flag in quality report).
- `EvidenceMap.summary.claims_with_evidence / claims_total >= 0.8` (target coverage).

## Validation Rules

- Reject (400/`VALIDATION_FAILED`) if `claims` empty or malformed.
- Reject if `sources` missing required fields (`id`, `title`).
- Drop low-confidence links (`relevance < min_score`).
- Deduplicate identical claim→source pairs when `policies.deduplicate=true`.

## Scoring Logic

- **Relevance** computed from text similarity (embedding cosine) + ontology overlap.
- **Strength** derived from publication type: meta-analysis>RCT>animal>in-vitro.
- **Confidence** = min(relevance, strength) × adjustment for recency and population match.

## Errors

- `VALIDATION_FAILED` — malformed claims or missing fields.
- `NO_MATCH_FOUND` — no evidence above threshold.
- `EXTERNAL_SOURCE_ERROR` — PubMed or other API failure.
- `INTERNAL` — unexpected error.

## Flags & Quality

- `low_confidence_links` — bindings where `confidence < 0.4`.
- `unlinked_claims` — claims without any supporting evidence.
- `duplicate_sources` — same source cited by multiple claims excessively (>N).

## Determinism

- Given identical `EvidenceLinkRequest`, `link_evidence` must produce identical `EvidenceMap` (excluding timestamps and source_score rounding).

## Persistence (Side Effects)

- Store: `evidence_map(id, memo_id, json, created_at)`.
- Maintain source cache keyed by `citation_id` for deduplication across memos.

## Security & Privacy

- Strip any PHI from evidence texts; store only bibliographic metadata.

## Versioning

- **Contract version:** `1.0.0`
- Additive-only schema changes in minor versions; breaking changes require new Agent ID (e.g., `A6b`).
