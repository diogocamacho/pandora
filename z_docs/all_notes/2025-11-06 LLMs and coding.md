## 🧠 Note

A post by Claus Wilke [here](https://blog.genesmindsmachines.com/p/llms-excel-at-programminghow-can?utm_source=post-email-title&publication_id=5419410&post_id=177950065&utm_campaign=email-post-title&isFreemail=true&r=2uxm8r&triedRedirect=true&utm_medium=email)


This is a great post where Claus talks about the 2 kinds of skillsets for programming and how LLMs are great at one, but not the other. The 2 skillsets:

- The first is being able to reason deeply and innovatively about algorithms, data structures, or software architecture. **This is the one LLMs lack**. 
- The second is being able to read, process, and memorize large amounts of API documentation, tutorial materials, and other existing code examples. **This is the one LLMs excel at**.

This is a great framing and is in line with how I think about it. I don't know Python, but LLMs are great at writing code in Python, so I don't really care if all the code is in Python, because I can read it and understand it. 

The things that the LLMs are not great at, in comp bio and ML, is the the reasoning and how to solve the problem in creative ways. I have lost countless hours in correcting the behavior of Cursor to do the analysis the way I want it to. It's fascinating how bad it is at reasoning a biology problem and put it into computational terms.
