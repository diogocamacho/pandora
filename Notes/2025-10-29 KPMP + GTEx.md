## 🧠 Note

I need to create a workflow for this analysis, so that I can define the specificities of kidney cell genes against all other cellular backgrounds.

# Objective

Build a **pan-tissue healthy reference** from GTEx scRNA-seq plus **healthy kidney scRNA-seq** from KPMP. Quantify gene **kidney cell-type specificity** against all other human cell types. Then map **KPMP disease scRNA-seq** (CKD, AKI) into that reference for abundance and state shifts.

# 1) Inputs and QC

**Datasets.**  
GTEx. all healthy scRNA-seq across tissues.  
KPMP healthy. kidney scRNA-seq only.  
KPMP disease. CKD and AKI scRNA-seq.

**Gene space.** Use Ensembl IDs. Keep protein-coding. Drop MT and rRNA for specificity metrics.

**Per-cell QC.**

- Min genes per cell. pick by dataset quantiles. e.g., >1,000 genes if 10xv3-like.
- Max mito fraction. dataset-specific, usually <10–15%.
- Complexity outliers. filter by log(UMIs) vs log(genes) residuals.
- Doublets. Scrublet or DoubletFinder. Keep a donor-level record of removed cells.
- Ambient RNA. SoupX or CellBender on each library.

# 2) Normalization and integration

- Keep **raw counts** for modeling.
- Create normalized views for diagnostics. `log1p(CPM)` or SCTransform per library.
- **Latent model** on raw counts. scVI or scANVI with batch covariates = donor, site, chemistry, tissue. Train **only on healthy data** first. This yields a pan-tissue healthy manifold with coherent cell types across organs.

# 3) Annotation of the healthy reference

- Cluster in latent space. multi-resolution Leiden.
- Annotate with a hierarchical scheme. major class → tissue → fine cell type. Use a curated marker panel and label transfer checks between GTEx and KPMP healthy kidneys.
- Lock these labels. This is your **frozen healthy reference**.

# 4) Gene specificity. definitions that do not lie

Work at **cell type × donor** granularity to avoid cell-count bias.

**Pseudo-bulk by donor.** For gene _g_, cell type _c_, donor _d_. sum counts across cells of _(c,d)_ then TMM or DESeq2 size factors. Compute `μ_{g,c,d} = log1p(CPM)`.

**Per-type expression.** `μ̄_{g,c} = mean_d μ_{g,c,d}` with donors from both GTEx kidney and KPMP healthy kidney.  
**Background distribution.** All other cell types from **all GTEx tissues**.

**Primary specificity metric.** τ (tau).  
`τ_g = Σ_c (1 − μ̄_{g,c}/max_c μ̄_{g,c}) / (K−1)` in [0,1]. High means specific.  
**Secondary metrics.**

- Gini on `{μ̄_{g,c}}`.
- AUROC for each c. classify donor-level pseudo-bulks of c vs non-c. report AUROC and log2FC = `μ̄_{g,c} − mean_c’≠c μ̄_{g,c’}`.
- “Pan-tissue leakage score”. max expression of g in **non-kidney** cell types. penalize markers that light up in liver macrophages, airway epithelium, etc.

**Stability checks.**

- Leave-one-dataset-out. recompute metrics excluding KPMP healthy or excluding GTEx kidney. retain genes with stable rank.
- Cross-platform sensitivity. flag genes with heavy 3’ UTR bias or chemistry sensitivity.

# 5) Summarization choice. mean or sum

- For **specificity**, do **donor-level pseudo-bulk means**. This treats donors as replicates. Robust, avoids over-weighting cell-rich donors.
- For quick exploration, cell-level **mean of normalized counts** per type is fine. Do not use raw **sums** for specificity. sums conflate abundance with expression.

# 6) Filtering even if “processed”

Yes. Apply light, explicit filters before specificity and mapping.

- Remove lowly expressed genes. keep genes expressed in ≥5% of cells in at least one kidney cell type **or** with μ̄ above a minimal threshold in any type.
- Exclude MT, rRNA, Ig/TCR for the marker set. keep them for QC plots only.
- Retain donor metadata. all downstream tests are donor-aware.

# 7) Bring in disease as a projection. not as training signal

- Use **scArches** to map KPMP CKD and AKI scRNA-seq into the frozen healthy manifold.
- **Cell-type assignment.** nearest-neighbor in latent space with posterior class probs. keep an “unassigned/novel” bucket to catch disease-only states.
- **Differential abundance.** per donor compositional model. scCODA or Dirichlet-multinomial regression on type proportions, controlling for age, sex, site if available.
- **Within-type differential expression.** donor-level pseudo-bulk. edgeR or DESeq2. contrasts = disease vs healthy donors mapped to the same type. report effect sizes with shrinkage.
- **State gradients.** within each kidney type, run diffusion pseudotime or VAM scoring for fibrosis, EMT, hypoxia modules to quantify state shifts without relabeling types.

# 8) Diagnostics that keep you honest

- **Batch mixing.** LISI or kBET should improve after integration.
- **Type purity.** silhouette by type should remain high.
- **Canonical markers.** NPHS1 in podocytes. AQP2 in collecting duct principal. SLC5A2 in proximal tubule. SLC12A1 in TAL. If these fail, revisit variable genes or chemistry covariates.
- **Pan-tissue leakage.** ensure kidney markers do not light up in airway or gut epithelial types in GTEx. prune offenders from the “kidney-specific” panel.

# 9) Deliverables

- **Healthy reference object.** AnnData or Seurat. latent space. locked labels.
- **Specificity table.** gene. best cell type. τ. AUROCs per type. log2FC to second-best type. pan-tissue leakage score. stability flags.
- **Disease report.** per donor and cell type. abundance change. DE genes. pathway scores. novel states.

# 10) Practical defaults

- scVI params. n_latent 30–50. library size covariate on. max epochs 2,000 with early stop.
- Variable genes. union of per-dataset HVGs then intersect across datasets. floor at 3k.
- Multiple testing. Benjamini–Hochberg per contrast. also apply an effect-size floor. e.g., |log2FC| ≥ 0.5 at donor-pseudo-bulk level.
- Reproducibility. freeze random seeds. write out the mapping from cluster IDs to ontology terms.

This workflow gives you kidney-cell markers that are **specific against the whole human single-cell background**, not just within kidney. It also cleanly separates disease **composition** from **state**, which is where most false positives creep in.
