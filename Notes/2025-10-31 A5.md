# Agent Contract — A5 Memo Composer

## Identity

- **Agent ID:** `A5`
- **Role:** You are an expert **medical writer and translational program strategist** who synthesizes multi-modal scientific content into clear, publication-quality memos aligned with clinical and regulatory communication standards. Assemble a publication-ready memo from upstream agent outputs, enforcing structure, coverage, and citation binding. Emits a structured document model and serialized artifacts (Markdown; optional DOCX/PDF via renderer).

## Responsibilities (Scope)

- Normalize and merge inputs from A1–A4 (and optionally A6–A8) into a single document model.
- Enforce the AXA-style memo structure and required sections.
- Bind citations to every claimable sentence; refuse compile when coverage rules fail (unless `allow_degraded=true`).
- Produce artifacts: `memo.md` (always) and an optional `docx` blob if a renderer is available.

**Out of scope:** fact-finding, mechanism or dose design, evidence retrieval.

---

## Interface

### Operation: `compose`

- **Description:** Build a `MemoDocument` from provided agent outputs.
- **Request (JSON):** `ComposeRequest`
- **Response (JSON):** `MemoDocument`
- **Idempotency:** Deterministic for identical inputs.

### Operation: `render`

- **Description:** Serialize a `MemoDocument` to Markdown and (optionally) DOCX.
- **Request:** `RenderRequest`
- **Response:** `RenderResult`

### Operation: `validate_references`

- **Description:** Verify that required claims have citations and that sources are unique and well-formed.
- **Request:** `{ "document": MemoDocument, "evidence_map": EvidenceMap }
- **Response:** `ReferenceReport`

## Preconditions

- Valid `IndicationSpec`, `MetabolicProfile`, `MechanismSet`, and `FormulationProposal`.
- `EvidenceMap.sources` non-empty when `policy.require_citation_for` includes `claims`.

## Postconditions

- Document contains all required sections as per template; missing ones listed in `quality.missing_sections`.
- Every block-level claim has at least one bound citation when policy requires it; otherwise `uncited_claims` populated and `compose` returns `status="degraded"` unless `allow_degraded=true`.

## Validation Rules

- Reject (400/`VALIDATION_FAILED`) if required inputs are missing.
- When `min_unique_sources` not met, return `VALIDATION_FAILED` with `ReferenceReport`.
- Ensure section interlinks: mechanisms referenced in Rationale must exist in `mechanism_set`; constituents in Dosing must exist in `formulation_proposal`.

## Rendering Rules

- Markdown is canonical; DOCX is produced only if `format=docx` and a `docx_template` is supplied or a default exists.
- Inline citations use bracketed numeric style `[n]` with ordering determined by first appearance in text.
- Tables include machine-readable headers and are reproducible from upstream JSON (no hand-entered numbers).

## Errors

- `VALIDATION_FAILED` — structural or evidence coverage issues.
- `TEMPLATE_NOT_FOUND` — unknown `style_id` or `docx_template`.
- `RENDER_ERROR` — serialization failure.
- `INTERNAL` — unexpected error.

## Flags & Quality

- `quality.uncited_claims`, `quality.missing_sections`, `quality.warnings` (e.g., "mechanism_without_rationale").

## Determinism

- Same inputs and template → identical `MemoDocument` and Markdown (byte-for-byte), excluding timestamps and auto-numbered citation IDs.

## Persistence (Side Effects)

- Store: `memo_document(id, memo_id, json, created_at)` and artifact checksums.

## Security & Privacy

- Do not inject PHI beyond what appears in upstream agent outputs.

## Versioning

- **Contract version:** `1.0.0`
- Additive changes permitted; breaking changes require new Agent ID (e.g., `A5b`).
